# 1.2 提示词工程

如果上一章讲的是"AI是什么"，这一章讲的就是"怎么跟AI说话"。

听起来挺简单的对吧？打字谁不会啊。但你肯定有过这种经历：同样一个问题，你问ChatGPT得到的回答平平无奇，别人问出来的结果让你惊呼"这也行？"。差距就在提示词上。

提示词工程（Prompt Engineering）是当前AI时代**每个人都必须掌握的核心技能**，不夸张地说，它决定了你能从AI身上榨出多少价值。好消息是，它不需要任何编程基础，纯靠自然语言就能搞定。

## 基础提示结构：四个要素

一个好的提示词通常包含四个部分，你可以把它想象成给实习生布置任务：

**角色设定：** 告诉AI它是谁。"你是一位有10年经验的Java后端工程师"比"帮我写代码"效果好得多。给了角色，AI就有了回答问题的视角和专业度基准线。

**任务描述：** 你到底要它干什么。这部分越具体越好。"帮我写一篇文章"是模糊的，"帮我写一篇800字的产品发布公告，面向企业客户，重点突出安全性和性能提升"就清晰多了。

**上下文信息：** 背景、参考资料、限制条件。AI不知道你的项目背景、你的用户是谁、你们公司的风格是什么样的。你给的背景越充分，它的输出就越贴合你的需求。

**输出格式约束：** 你希望结果长什么样。Markdown表格？JSON格式？分点列出？中文还是英文？不说清楚，AI就自由发挥，大概率不是你想要的格式。

来看一个完整的例子：

```
你是一位资深的用户体验设计师，擅长B端产品设计。

我正在设计一个企业内部的项目管理系统，目标用户是30-45岁的项目经理，
他们通常同时管理3-5个项目，最大的痛点是跨项目的资源冲突和进度追踪。

请帮我分析这类用户在"查看项目总览"这个场景下的核心需求，
并给出Dashboard页面的信息架构建议。

输出格式：
1. 用户核心需求（3-5条）
2. Dashboard推荐的信息模块（每个模块说明名称、作用和优先级）
3. 用表格呈现信息模块
```

你看，角色、任务、上下文、格式，四个部分都有了。这样的提示词出来的结果，和你直接说"帮我设计一个Dashboard"，完全是两个级别。

## 进阶技巧：让AI思考得更好

掌握了基础结构，再来看几个让输出质量再上一个台阶的技巧。

**Few-shot Prompting（少样本提示）。** 直接告诉AI"按某种风格写"很抽象，但如果你给它几个例子，它立刻就懂了。比如你要它帮你写JIRA的ticket描述，与其解释格式要求，不如直接贴两个你以前写得好的ticket作为范例，然后说"按同样的风格和结构写"。人类学东西靠模仿，AI也一样。

**Chain-of-Thought（思维链）。** 遇到复杂问题的时候，加一句"请一步一步地思考"或者"请先分析再给出结论"，效果会显著提升。原理很直观：当你要求AI把推理过程写出来的时候，它在生成每一步的过程中实际上在为下一步提供更好的上下文。就像你做数学题，在草稿纸上一步步写和直接心算，准确率是不一样的。

**Self-Consistency（自一致性）。** 对于同一个问题，让AI回答多次，然后取最一致的那个答案。这在需要准确性的场景下特别有用。你可以说"请给出三种不同的分析思路，然后比较它们的优劣，选出最合理的一种"。

## 提示链：把大象装进冰箱

复杂任务直接一个提示词搞定，效果往往不好。更好的做法是**拆成多步**，每一步的输出作为下一步的输入，这就是提示链（Prompt Chaining）。

举个实际例子。假设你要AI帮你写一份竞品分析报告：

- **第一步：** "请帮我列出XX领域的5个主要竞品，每个给出一句话定位描述。"
- **第二步：** "针对以上5个竞品，请从产品功能、定价策略、目标用户三个维度进行对比分析。"
- **第三步：** "基于上面的分析，请总结我们产品的差异化优势和需要补齐的短板。"
- **第四步：** "请把以上所有内容整合成一份结构化的竞品分析报告，包含摘要、详细分析和行动建议。"

每一步都聚焦在一个具体的子任务上，AI处理起来更得心应手，你也有机会在中间环节纠偏。如果最终结果有问题，你能精确定位是哪一步出了偏差，而不是整个推倒重来。

## 调试与优化：提示词也需要迭代

没有人能一次写出完美的提示词。跟写代码一样，提示词也需要调试和迭代。

**看哪里不满意，针对性地改。** AI的输出太笼统？加更多具体约束。格式不对？明确指定格式要求。语气不合适？在角色设定里加上语气要求。漏掉了某些方面？在任务描述里补上。

**用否定句划定边界。** 有时候告诉AI"不要做什么"比"要做什么"更有效。"不要使用技术术语，目标读者是非技术人员"、"不要泛泛而谈，每个建议都必须可直接执行"。

**让AI自己反思。** 一个很好用的技巧：在AI给出回答之后，追问"你觉得这个回答有什么不足？如果重新回答，你会怎么改进？"。AI的自我反思能力往往能帮你发现盲点。

**保存你的好提示词。** 当你调试出一个效果很好的提示词模板，存下来。下次遇到类似任务直接复用和微调，效率会高很多。慢慢地你会积累出一个属于自己的提示词库，这就是你的AI生产力资产。

最后再强调一点：**提示词工程的核心不是记住某些固定的"咒语"，而是培养一种清晰表达需求的思维方式。** 你能把需求想清楚、说清楚，AI就能给你好结果。说到底，和AI沟通的能力，本质上就是结构化思考和表达的能力。这个能力不光对AI有用，对你跟人协作也同样有价值。
