# 2.5 AI辅助UI设计实战

上一节我们聊了怎么用AI生成视觉素材。这一节回到UI设计师最核心的工作——界面设计本身。AI在这个领域的能力进展很快，虽然还不能替代设计师做完整的UI设计，但在好几个环节已经能实实在在地帮你省时间。

## 用AI快速生成UI概念

很多时候，设计工作的瓶颈不在画图，而在想清楚"这个界面应该长什么样"。产品经理给你一段需求描述，你需要先在脑子里把它转化成界面结构，然后再动手画。这个"从文字到视觉"的转化过程，AI可以帮你加速。

**第一步：用ChatGPT或Claude梳理界面结构**

假设你接到一个需求："设计一个健身App的训练计划页面，用户可以查看本周的训练安排、今天的训练详情、以及训练进度统计。"

直接画图之前，先跟AI聊一下：

```
我在设计一个健身App的训练计划页面。用户需求包括：
1. 查看本周训练安排（周一到周日）
2. 查看今天训练的详细内容（动作名称、组数、次数）
3. 查看训练进度（本周完成率、连续打卡天数等）

请帮我梳理这个页面的信息架构和布局建议。考虑移动端的屏幕空间限制，
给出你认为最合理的信息优先级排列。
```

AI会给你一个结构化的布局建议，包括各个模块的优先级、大致的空间分配、交互逻辑。你不一定完全按它说的做，但它能帮你快速理清思路，特别是当你同时做好几个项目、脑子里装不下那么多东西的时候。

**第二步：用AI工具把概念可视化**

文字方案确认后，你可以用AI工具直接生成界面草案。

## 用v0.dev生成UI界面

v0.dev是Vercel推出的AI UI生成工具，我个人觉得它是目前最实用的AI UI生成方案之一，因为它生成的不只是图片，而是真正可以运行的React代码。

操作流程：

1. 打开 v0.dev，在输入框中描述你想要的界面
2. 描述越具体越好。比如：

```
一个健身App的训练计划页面，移动端设计：
- 顶部显示今天日期和问候语
- 下方是一个横向滑动的本周日历，当天高亮显示
- 主体区域展示今天的训练内容，卡片式布局，每张卡片包含动作名称、
  动作示意图标、组数x次数
- 底部固定一个"开始训练"的主按钮
- 整体风格：深色主题，强调色用亮绿色
```

3. AI会生成一个可交互的界面预览，你可以直接在浏览器中查看效果
4. 如果某个部分不满意，在对话中继续描述你想修改的内容，AI会迭代更新

v0.dev生成的结果通常在布局合理性上表现不错，配色和间距也过得去。它的定位不是取代你的设计，而是帮你在5分钟内得到一个可以讨论的起点。拿着这个起点跟产品经理对齐，比拿着一段文字描述对齐效率高太多了。

## 用Galileo AI生成完整页面

Galileo AI的定位更接近传统设计工具，它生成的是设计稿而不是代码。

1. 进入 Galileo AI，选择你要设计的平台（iOS/Android/Web）
2. 输入页面描述，可以比较自然地写，比如"a fitness app workout tracking screen with dark theme"
3. AI会生成多个设计方案供选择
4. 选定一个方案后可以进入编辑模式，对细节进行调整
5. 可以导出为Figma格式，继续在Figma中精修

Galileo AI的优势是它对UI设计规范的理解比较好，生成的间距、字号、组件比例通常符合平台设计指南。劣势是可定制程度有限，复杂的交互场景它处理不了。

## AI辅助组件设计

除了整页设计，AI在单个组件的设计上也能帮忙。这个场景可能更贴近你的日常工作。

比如你需要设计一个搜索组件，要支持搜索建议、历史记录、筛选标签。你可以这样跟AI沟通：

```
我需要设计一个电商App的搜索组件，要求：
- 搜索框获得焦点后展开，显示搜索历史（最近5条）和热门搜索
- 输入文字后实时显示搜索建议，按相关度排序
- 搜索框下方有横向滑动的筛选标签（品类、价格区间、品牌等）
- 需要考虑空状态（没有搜索历史时怎么显示）

请给出这个组件在不同状态下的设计方案描述，包括：
默认状态、获得焦点状态、输入中状态、有结果状态、无结果状态。
```

AI会给你一个完整的状态清单和每个状态的布局建议。你可能会发现它提到了一些你没考虑到的边界情况，比如搜索词高亮、输入法键盘弹起后的布局调整。这就是AI作为"设计思维补充"的价值——它不会忘记考虑edge case。

## AI驱动的设计走查

设计完成后的走查环节，AI也能发挥作用。把你的设计稿截图发给ChatGPT或Claude，让它帮你做检查：

```
请对这张App界面设计稿进行设计走查，检查以下方面：
1. 信息层次是否清晰，视觉重心是否明确
2. 间距和对齐是否一致
3. 文字可读性（字号、对比度）
4. 交互元素（按钮、链接）是否容易识别和点击
5. 是否考虑了无障碍设计（色彩对比度、触控区域大小）
6. 与iOS/Material Design规范的一致性
```

AI会给你一份相当详细的检查报告。它不能替代真正的用户测试，但能帮你在正式评审前捕捉到一些明显的问题。我自己的经验是，AI特别擅长发现一致性方面的问题——比如同一个页面用了三种不同的间距、某个按钮的圆角跟其他按钮不一样。这些细节人眼看多了容易麻木，AI不会。

## 实际工作流示例

把上面这些串起来，一个完整的AI辅助UI设计工作流是这样的：

**需求分析阶段**（10分钟）：把PRD里的需求描述丢给Claude，让它帮你梳理信息架构、列出需要考虑的状态和边界情况。这一步产出的是一份结构化的设计需求清单。

**快速原型阶段**（15分钟）：把整理好的需求描述输入v0.dev或Galileo AI，生成2-3个方向的界面草案。截图保存，作为后续讨论的素材。

**方案对齐阶段**（20分钟）：拿着AI生成的草案跟产品经理过一遍，快速确认方向。因为有可视化的东西可以看，讨论效率比纯靠语言描述高很多。

**精细设计阶段**（这一步时间因项目而异）：在Figma中基于确认的方向做正式设计。这一步AI的角色退到辅助位置——帮你写占位文案、生成模拟数据、处理图片素材。核心的设计决策还是你来做。

**设计走查阶段**（10分钟）：设计完成后把稿子截图给AI做一遍检查，记录下它提到的问题，该修的修。

整个流程下来，AI参与的部分加起来大概省了1-2小时的工作量。这个时间不是靠加速画图省出来的，而是靠减少"从零开始想"和"反复沟通对齐"省出来的。设计本身的质量还是取决于你这个设计师的水平，AI只是帮你把精力更多地集中在真正需要创造力的部分。
