# 2.6 AI辅助设计交付

设计做完了，接下来是交付。如果你干过几年设计就知道，交付环节消耗的时间经常比你预期的多——切图要导各种尺寸、标注要写清楚各种间距和色值、设计说明文档要让开发看得懂、评审要准备多版本的对比展示。这些事情繁琐但不能省，而且每一项都很适合让AI来搭把手。

## AI辅助切图与标注

先说切图。现在大多数团队已经在用Figma之类的工具做设计，Figma本身的开发模式（Dev Mode）已经能自动生成大部分标注信息了。AI在这一层能做的更多是补充和优化。

**自动生成多尺寸适配**：如果你需要为同一组素材导出不同尺寸（1x/2x/3x、不同设备分辨率），可以用AI帮你批量处理。具体操作是把你的切图规范告诉ChatGPT或Claude：

```
我有一组App图标需要导出，原始尺寸是1024x1024px。请帮我列出iOS和Android
平台需要的所有尺寸规格，以及对应的文件命名规则。
```

AI会给你一份完整的尺寸清单，包括iOS的@2x/@3x、Android的mdpi到xxxhdpi，以及App Store和Google Play需要的各种尺寸。这份清单你可以直接拿来作为批量导出的依据。

**智能标注补充**：Figma的自动标注有时候不够细致，特别是对一些隐含的设计逻辑（比如"这个间距是8px的倍数体系""这个阴影随着层级递增"），开发看不出来。你可以把设计稿截图加上已有的标注，发给AI：

```
这是一个卡片组件的设计稿，请帮我检查标注是否完整，
特别关注以下方面：
- 间距体系是否一致（是否遵循8px网格）
- 字号层级关系
- 颜色值是否都标注了（包括透明度）
- 阴影和圆角参数
- 可点击区域的实际大小
如果有遗漏，请指出哪些地方需要补充标注。
```

## 用AI生成设计说明文档

这是我觉得AI最能直接产出价值的交付环节。写设计说明文档是大多数设计师最不想做的事情——但对开发来说这份文档太重要了。

**生成组件级设计说明**：假设你设计了一个自定义的底部导航栏，可以这样让AI帮你写文档：

```
请帮我编写一个底部导航栏组件的设计说明文档，面向前端开发工程师。
包含以下信息：

组件名称：BottomNavBar
包含5个Tab项：首页、发现、+（发布）、消息、我的
设计参数：
- 导航栏高度：56px（不含安全区域），底部安全区域跟随系统
- 背景色：#FFFFFF，上边框：0.5px solid #E5E5E5
- 图标尺寸：24x24px，选中态为品牌色#FF6B35，未选中态为#999999
- 文字字号：10px，与图标间距4px
- 中间的发布按钮突出显示，圆形，直径48px，背景为品牌色
- 点击态：图标缩放至90%，200ms ease-out动效

请按照开发友好的格式组织这份文档，包括状态说明、交互说明和边界情况。
```

AI会生成一份结构清晰、参数明确的技术文档。你过一遍确认参数没错，就可以直接发给开发了。这比你自己从零开始写至少省半小时。

**生成交互说明**：对于复杂的交互流程，AI也能帮你把设计意图翻译成开发能理解的语言：

```
请帮我编写以下交互流程的技术说明：

场景：用户下拉刷新商品列表
1. 用户在列表顶部下拉，出现刷新动画
2. 下拉超过60px时松手触发刷新，不超过则回弹
3. 刷新中显示loading动画，持续时间取决于接口返回
4. 刷新成功：新数据从顶部插入，列表回到顶部
5. 刷新失败：显示Toast提示"刷新失败，请稍后重试"
6. 列表为空时：显示空状态页面，包含插图和"暂无内容"文案

请用前端工程师熟悉的方式描述，包括动画参数、时序关系和异常处理。
```

## AI辅助设计评审

设计评审的核心痛点是什么？是你需要在短时间内让评审者理解你的设计决策，同时展示你考虑过的替代方案。AI可以在两个方面帮你。

**快速生成多版本方案对比**：假设你为一个登录页面做了设计，你认为方案A最好，但你知道评审时一定有人会问"为什么不用XXX的方式"。你可以提前准备：

```
我设计了一个App登录页面，最终选择了方案A（简洁表单式，邮箱+密码）。
请帮我列出常见的登录页面设计方案，并分析各自的优缺点，
这样我在评审时可以解释为什么选择了当前方案：
- 方案A：传统表单式（邮箱+密码）
- 方案B：手机号+验证码
- 方案C：社交账号一键登录为主
- 方案D：生物识别（Face ID/指纹）为主
针对我们的场景（面向国内用户的生活服务类App，核心用户25-40岁），
请分析每个方案的适合度。
```

AI给出的分析可以直接作为你设计决策的支撑材料。评审时有人提问，你可以清楚地解释"我考虑过B方案，但因为XXX原因选择了A"。

**生成评审展示文案**：你还可以让AI帮你把设计思路组织成评审展示的结构：

```
请帮我组织一份设计评审的展示大纲，项目是一个电商App的购物车改版。
我的设计改动包括：
1. 将优惠券入口从底部移到商品列表上方
2. 增加了商品分组（按店铺）
3. 将"猜你喜欢"推荐区从购物车移除
4. 新增了库存不足和失效商品的处理方式

请按"问题-方案-依据"的结构来组织，让评审者能快速理解每个改动的原因。
```

## 用AI做设计到开发的沟通桥梁

设计师和开发之间的沟通障碍，很多时候不是因为设计不好或者开发不用心，而是双方用的"语言"不一样。设计师说"这里要有一个呼吸感的动效"，开发听完一脸问号。AI可以在中间做翻译。

**把设计语言翻译成技术语言**：

```
我需要向前端开发描述以下动效：
用户点击收藏按钮时，心形图标先快速放大到120%，然后回弹到100%，
同时颜色从灰色变为红色，按钮周围散发出几个小红心粒子向四周飘散消失。

请把这段描述翻译成前端开发能直接实现的技术方案，包括：
- 动画时间线和缓动函数
- CSS/JS实现思路
- 关键帧参数
```

AI的输出：
```
动画时间线（总时长约600ms）：

1. 图标缩放（0-300ms）
   - 0-150ms: scale(1) → scale(1.2), ease-out
   - 150-300ms: scale(1.2) → scale(1), ease-in-out (弹性回弹)

2. 颜色变化（0-200ms）
   - color: #CCCCCC → #FF4757, ease-in

3. 粒子效果（100-600ms）
   - 生成6-8个圆形粒子，直径4px，颜色#FF4757
   - 从按钮中心向四周以随机角度扩散
   - 每个粒子独立动画：
     - 位移：从中心向外移动30-50px（随机距离）
     - 透明度：1 → 0
     - 缩放：1 → 0.5
     - 时长：400-500ms，ease-out

建议实现方式：
- 图标动画：CSS transform + transition
- 粒子效果：Canvas或CSS animation with @keyframes
- 如使用Lottie，可导出为JSON动画文件直接使用
```

你把这个发给开发，他就不用再猜你说的"呼吸感"到底是什么参数了。

**生成设计走查对照清单**：开发完成后你需要做设计走查，确认实现跟设计稿一致。让AI帮你生成一份检查清单：

```
请根据以下页面设计要素，生成一份前端实现的设计走查清单：
页面：商品详情页
关键要素：顶部轮播图、价格区域、规格选择、加入购物车按钮、
商品描述、评价区域、底部操作栏

每个要素需要检查：尺寸、间距、颜色、字体、层级关系、
点击区域、滑动行为、加载状态、空状态。
```

AI会生成一份逐项对照的清单，你拿着这份清单对比实际实现，遗漏的问题会少很多。走查完之后再把需要修改的点统一整理给开发，比想到一个提一个效率高，开发也不会觉得你在"挤牙膏"。

说到底，AI在设计交付环节的价值不是帮你做设计决策，而是帮你把已经做好的设计决策高效、准确、无歧义地传达给下游。设计师最宝贵的时间应该花在思考和创造上，而不是花在写文档和对参数上。让AI来处理这些，你专注于真正需要你的部分。
