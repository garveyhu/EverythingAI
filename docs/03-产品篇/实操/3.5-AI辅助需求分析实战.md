# 3.5 AI辅助需求分析实战

前面讲了不少理论，这一节开始动手。我会把产品经理做需求分析时最常见的三个场景拿出来，一步一步演示怎么用AI来提效。每个场景都会给完整的Prompt模板，你可以直接复制过去改改就用。

## 场景一：用AI做竞品分析

竞品分析是产品经理的基本功，但也是最耗时间的工作之一。你得搜集信息、整理功能对比、分析优劣势、写报告——一整套下来至少两三天。用AI可以把这个过程压缩到几个小时。

### 步骤1：信息准备

在让AI分析之前，先把你能搜集到的竞品信息整理好。包括但不限于：竞品的官网介绍、功能页面截图或文字说明、定价页面信息、用户评价（G2、Capterra、App Store等平台）、新闻报道或行业分析。

信息不需要很完美，但**你给AI的输入质量直接决定输出质量**。一份空洞的竞品信息只会得到空洞的分析。

### 步骤2：使用结构化Prompt进行分析

下面是一个经过反复调试的竞品分析Prompt模板：

```
你是一位资深产品分析师。请根据我提供的信息，对以下竞品进行结构化分析。

## 我的产品
- 产品名称：[你的产品名]
- 所属领域：[比如：项目管理SaaS]
- 目标用户：[比如：50-500人规模的互联网公司]
- 核心定位：[一句话描述你的产品价值主张]

## 待分析竞品
竞品名称：[竞品名]

以下是我收集到的竞品信息：
---
[把搜集到的信息贴在这里，包括功能介绍、定价、用户评价等]
---

请按以下框架输出分析结果：

1. **产品定位分析**：这个竞品的核心定位是什么？目标用户是谁？跟我的产品有多大程度的重合？

2. **功能对比**：列出竞品的核心功能模块，并与我的产品做对比。用表格形式呈现，标注"我们有/我们没有/我们更强/我们更弱"。

3. **差异化优势**：竞品做了什么我们没做的事情？其中哪些是真正有价值的差异化，哪些是噱头？

4. **定价策略分析**：竞品的定价模型是什么？跟我们相比贵还是便宜？价格差异背后的逻辑是什么？

5. **用户口碑摘要**：根据用户评价，这个竞品被夸最多的3个点和被吐槽最多的3个点分别是什么？

6. **对我们的启示**：基于以上分析，给出3-5条对我们产品有参考价值的建议。明确标注哪些是"建议跟进"，哪些是"了解即可，暂不行动"。

请用务实的语气输出，不要泛泛而谈，每个观点尽量有具体的功能或数据支撑。
```

### 步骤3：追问与深挖

第一轮输出通常是框架性的，你需要针对感兴趣的点继续追问。比如：

```
你提到竞品X的协作功能做得比我们好，请具体说明：
1. 他们的协作功能具体包含哪些子功能？
2. 跟我们的协作功能相比，核心差距在哪？
3. 如果我们要补齐这个差距，你估计需要多大的开发投入（按功能复杂度分S/M/L）？
```

### 步骤4：生成最终报告

分析完所有竞品后，让AI帮你汇总：

```
请根据前面的分析，生成一份竞品分析总结报告。要求：
1. 用竞争格局图的形式展示各竞品的定位关系（用文字描述，我后续会转成图表）
2. 汇总我们产品的核心优势和主要差距
3. 给出功能优先级建议：最应该投入资源去做的3件事
4. 整份报告控制在1500字以内，适合给管理层汇报
```

## 场景二：用AI做用户调研分析

做完用户访谈或收完问卷后，面对一堆原始记录，整理分析又是一个大工程。AI可以帮你快速提取关键信息并形成结论。

### 处理访谈记录

假设你有一批用户访谈的文字记录（可以是录音转写的文本），使用以下Prompt：

```
你是一位用户研究分析师。我会提供一组用户访谈的原始记录，请帮我进行系统分析。

## 背景信息
- 产品：[产品名]
- 调研目的：[比如：了解用户在数据导出功能上的痛点和需求]
- 受访者画像：[比如：10位企业数据分析师，使用产品3个月以上]

## 访谈记录
---
[粘贴访谈记录，不同受访者用"受访者A/B/C..."区分]
---

请按以下结构输出分析：

1. **核心发现**（不超过5条）：跨受访者的共性发现，每条发现需注明有几位受访者提到了类似观点

2. **需求清单**：从访谈中提取的用户需求，按提及频率排序。每个需求包含：
   - 需求描述（一句话）
   - 用户原话引用（选最有代表性的1-2句）
   - 需求紧迫度评估（高/中/低，基于用户表述的情感强度）

3. **意外发现**：有没有什么发现是出乎意料的、或者只有少数人提到但很有价值的洞察？

4. **分析局限性**：这次分析可能存在什么偏差？比如样本是否有代表性问题、访谈引导是否可能影响了回答等。
```

### 处理问卷数据

如果是结构化的问卷数据，可以导出为CSV后上传给AI分析（注意脱敏），或者把统计结果贴进去：

```
以下是一份用户满意度调查的统计结果，共收到[X]份有效回复。

[粘贴问卷各题目的统计数据]

请帮我：
1. 找出满意度最高和最低的3个维度
2. 分析开放题回答中的高频关键词和情感倾向
3. 看看不同用户群体（按使用时长、公司规模等维度）的满意度是否有显著差异
4. 给出3条最值得关注的洞察和对应的行动建议
```

## 场景三：用AI生成用户故事和验收标准

需求分析的最终产出通常是用户故事（User Story）和验收标准（Acceptance Criteria）。这个环节AI能帮你做得又快又规范。

### 从需求描述生成用户故事

```
你是一位敏捷教练。请根据以下需求描述，生成规范的用户故事。

## 需求描述
[用大白话描述需求，比如：我们想在产品里加一个功能，让用户可以把项目报告一键导出为PDF格式，包含图表和数据表格，并且可以自定义报告的封面和页眉。]

## 产品上下文
- 产品类型：[项目管理SaaS]
- 目标用户角色：[项目经理、部门负责人]
- 已有相关功能：[目前支持在线查看报告，可以导出为Excel]

请输出：

1. **Epic描述**：一句话概括这个需求的整体目标

2. **用户故事列表**：拆分为多个独立的用户故事，每个用户故事遵循格式：
   "作为[用户角色]，我希望[功能描述]，以便[业务价值]"

3. **每个用户故事的验收标准**：使用Given-When-Then格式，覆盖正常流程和异常场景

4. **故事优先级建议**：哪些是MVP必须做的，哪些可以放到后续迭代

5. **技术风险提示**：从产品角度看，这个需求在实现时可能遇到什么技术挑战，需要提前跟开发沟通的点
```

### 实际输出效果

以"PDF导出"这个例子为例，AI通常会输出类似这样的内容：

> **用户故事1：** 作为项目经理，我希望将项目报告一键导出为PDF文件，以便离线分享给没有系统账号的客户。
>
> **验收标准：**
>
> - Given 用户在报告详情页，When 点击"导出PDF"按钮，Then 系统生成PDF文件并触发下载
> - Given 报告中包含图表，When 导出为PDF，Then 图表以高清图片形式嵌入PDF中
> - Given 网络连接中断，When 导出过程中断，Then 显示错误提示并允许重试

你拿到这个输出后，要做的是**审查和调整**，而不是直接复制粘贴。检查一下：用户角色对不对、业务价值描述是否准确、验收标准是否覆盖了你知道的边界情况。AI帮你搭好了框架，你来填入真正的业务判断。

### 一个提醒

用AI做需求分析，最大的风险是**看起来很专业但内容空洞**。AI特别擅长生成格式工整、逻辑通顺的内容，但如果你给它的输入缺乏真实的业务信息，它输出的就是"正确但没用"的内容。

所以每次使用之前，先问自己：我给AI的信息够不够具体？如果你自己觉得信息模糊，AI分析出来的东西也不会清晰。**AI是放大器，放大的是你输入的质量。**
