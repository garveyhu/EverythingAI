# 3.7 AI辅助原型与验证

产品经理日常有个痛点：脑子里有想法，但画不出来。你知道这个功能大概该长什么样，但要打开Figma从零开始画，一画就是半天，还不一定画得好。另一个痛点是验证——你做了一个方案，想快速知道用户会不会买账，但正经做用户测试周期又太长。

AI在这两件事上都能帮你。不是替代设计师，也不是替代真实的用户调研，而是**让你在方案早期就能快速形成可讨论的产出**。

## 场景一：用AI快速生成产品原型

### 工具选择

目前能帮产品经理快速生成原型的AI工具主要有这几个：

**v0.dev**（Vercel出品）。你用自然语言描述想要的界面，它直接生成可交互的前端代码和预览。适合生成Web端的界面原型，输出质量比较高，而且可以在线编辑和迭代。

**Galileo AI**。专门做UI设计稿生成的工具，输出的是高保真设计稿，风格比较现代。适合需要给团队或客户看"大概长什么样"的场景。

**Claude/ChatGPT + Artifacts**。直接在对话中让AI生成HTML页面或React组件，Claude的Artifacts功能可以即时预览。灵活度最高，适合快速验证交互逻辑。

我个人推荐的工作流是：**先用Claude/ChatGPT快速验证交互逻辑，再用v0.dev生成更精细的原型**。前者帮你想清楚"流程对不对"，后者帮你展示"体验好不好"。

### 用v0.dev生成原型的步骤

**第一步：准备你的描述。** v0.dev接受自然语言输入，但描述越具体，生成效果越好。一个好的描述应该包含：页面的整体布局、核心元素有哪些、交互行为是什么、风格偏好。

示例Prompt：

```
一个SaaS产品的数据仪表盘页面。

布局：
- 顶部是导航栏，包含产品Logo、搜索框、通知图标、用户头像
- 左侧是折叠式侧边栏，包含"概览""项目""团队""设置"四个菜单项
- 主内容区域分为上下两部分：上面是4个数据卡片（本月活跃用户、新增用户、付费转化率、NPS评分），下面是两个图表并排（左边是折线图展示过去12个月的用户增长趋势，右边是饼图展示用户来源渠道分布）

风格：
- 简洁现代的商务风格
- 主色调为深蓝色，辅助色为浅灰
- 数据卡片有轻微的阴影效果
- 图表使用柔和的渐变色

交互：
- 侧边栏可以点击折叠/展开
- 数据卡片hover时有轻微的放大效果
- 图表支持hover显示具体数值
```

**第二步：迭代调整。** 第一次生成的结果通常需要调整。你可以直接在v0.dev中用自然语言说"把数据卡片改成一行3个""左侧菜单加一个'报告'选项""整体配色换成绿色系"，它会在已有基础上修改。

**第三步：导出或分享。** v0.dev生成的原型可以直接分享链接给团队讨论，也可以导出代码交给前端开发参考。

### 用Claude生成交互原型

如果你更关注交互流程而不是视觉效果，Claude的Artifacts功能非常好用：

```
请帮我生成一个可交互的原型页面，使用HTML+CSS+JavaScript实现。

功能：一个AI智能搜索的交互流程

页面包含：
1. 搜索框，支持输入自然语言查询
2. 用户输入后，显示一个"AI正在分析..."的加载动画（模拟2秒延迟）
3. 加载完成后显示搜索结果：
   - 顶部是AI对查询意图的理解（比如"我理解你在找：XX相关的XX文件"）
   - 下面是结果列表，每个结果包含标题、摘要、相关度评分
   - 每个结果右侧有"采纳"和"不相关"两个按钮
4. 底部有一个"AI搜不到想要的？试试手动筛选"的链接

请用简洁的样式，可以用Tailwind CSS。重点是展示交互流程，不需要真实数据。
```

这个方法特别适合在需求评审会前快速出一个可演示的原型。比起静态线框图，一个能点击、能看到交互流程的原型，沟通效率高出很多。

## 场景二：AI辅助数据仪表盘搭建

产品经理经常需要搭建数据看板来监控产品指标。虽然最终的仪表盘需要用专业工具（比如Metabase、Grafana、公司内部BI平台），但在规划阶段，AI能帮你把"应该看哪些数据"和"怎么展示"想清楚。

### 步骤1：让AI帮你定义指标体系

```
我需要为我们的产品搭建一个产品经理日常使用的数据仪表盘。

产品信息：
- [产品名]，一个面向[目标用户]的[产品类型]
- 核心业务流程：[简述，比如：用户注册→创建项目→邀请成员→日常使用→付费升级]
- 目前的业务重点：[比如：提升新用户激活率和付费转化率]

请帮我设计仪表盘的指标体系：

1. 按"北极星指标 → 一级指标 → 二级指标"的层级结构来组织
2. 每个指标说明：定义、计算方式、数据来源、建议的查看频率
3. 指标之间的关联关系（哪个指标下降了应该去看哪个子指标排查）
4. 建议的仪表盘布局：哪些指标放在第一屏（最重要，每天看），哪些放在详情页（按需查看）
```

### 步骤2：生成可视化方案

拿到指标体系后，让AI帮你规划每个指标用什么图表呈现：

```
基于上面的指标体系，请帮我规划仪表盘的可视化方案：

1. 每个指标建议使用什么图表类型（折线图、柱状图、饼图、数字卡片、热力图等），并说明理由
2. 图表的布局排列建议（哪些放在一起方便对比）
3. 需要支持什么筛选维度（时间范围、用户群体、渠道等）
4. 有没有什么数据异常的自动预警建议（比如某指标下降超过X%时高亮提示）
```

这个输出你可以直接拿给BI团队或数据工程师，作为仪表盘搭建的需求文档。

## 场景三：用AI模拟用户反馈

这个场景有点"非主流"，但我用过很多次，效果出乎意料地好。

在方案还没上线、甚至还没开发的时候，你可以让AI扮演不同类型的用户，对你的方案给出反馈。这不能替代真实用户测试，但能帮你**在早期发现一些明显的问题**。

### 步骤1：定义用户角色

```
我需要你分别扮演以下三种用户，对我的产品方案给出反馈：

用户A - 小白用户：
- 30岁，市场专员，不太懂技术
- 使用我们产品3个月，主要用基础功能
- 对新功能的态度是"能用就行，别太复杂"

用户B - 重度用户：
- 35岁，项目总监，每天使用产品超过3小时
- 使用我们产品2年，几乎用过所有功能
- 对产品有很多改进意见，期望值很高

用户C - 决策者：
- 42岁，VP级别，不直接使用产品但负责采购决策
- 关注的是ROI和团队效率提升
- 看demo和报告做判断
```

### 步骤2：让AI以不同角色身份评审你的方案

```
以下是我计划上线的一个新功能方案：

[详细描述你的方案，包括功能说明、交互流程、使用场景]

请分别以用户A、用户B、用户C的身份，对这个方案给出反馈。每个角色的反馈包括：

1. 第一反应：看到这个功能介绍后的直觉感受
2. 最吸引我的点：哪个部分让我觉得"这个不错"
3. 最困惑或担心的点：哪个部分让我觉得"这我不太明白"或"这可能有问题"
4. 我会怎么使用：描述我实际使用这个功能的场景
5. 改进建议：如果能改一个地方，我最希望改什么

请用口语化的方式输出，像真实用户说话那样。不要太礼貌、不要太客观——真实用户的反馈是带感情色彩的。
```

### 步骤3：汇总分析

```
基于三位"用户"的反馈，请帮我做一个汇总分析：

1. 三个角色共同关注的问题是什么？（这大概率是真问题）
2. 哪些反馈只是个别角色的偏好？（可以记录但不紧急处理）
3. 基于这些反馈，我的方案最应该调整的1-2个点是什么？
4. 这次模拟分析可能有什么偏差？（提醒我后续还是要做真实用户验证）
```

我用这个方法帮自己发现过不少盲点。有一次我设计了一个AI推荐功能，自己觉得非常巧妙，但让AI扮演小白用户后，它指出"我不明白这些推荐是怎么来的，感觉有点creepy"——这个反馈提醒我在功能上加了一段解释文案，后来真实上线后用户果然反映"有解释就放心多了"。

当然，AI模拟的用户不是真实用户。AI扮演的角色偏"理性"，真实用户的反应可能更情绪化、更不可预测。所以这个方法的定位是**预检**——帮你在花大量资源之前，过滤掉最明显的问题。真正的验证还是要靠实际的用户测试。

## 这三个场景的共同逻辑

回过头看，AI辅助原型、辅助数据规划、模拟用户反馈，这三件事的共同逻辑是一样的：**降低验证成本，让你能更快地从"我觉得"走向"来看看"**。

以前你要花一周出一个原型、花几天搭一个数据看板、花两周做一轮用户测试。现在这些事情的"初版"都能在几个小时内完成。这意味着你可以在同样的时间里探索更多方向、测试更多假设、做出更有把握的决策。

这就是AI给产品经理带来的最大价值——不是替你做决策，而是**让你试错的成本变低了**。试错成本低了，你就敢于尝试更多可能性，最终做出的产品就有更大概率是对的。
