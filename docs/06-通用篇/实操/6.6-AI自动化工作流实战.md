# 6.6 AI自动化工作流实战

## 什么是工作流自动化

先从一个场景说起：你的团队每天收到客户反馈邮件，有人负责看邮件、分类、把重要的转发给对应负责人、把内容记录到表格里。这个过程完全是机械性的——判断规则是固定的，操作步骤是重复的。

工作流自动化就是把这类"触发条件明确、操作步骤固定"的流程交给软件自动执行。以前的自动化工具（比如Excel宏、邮件规则）只能做简单的条件判断；现在加上AI之后，自动化的能力边界大大扩展了——AI可以理解邮件内容、做语义分类、生成摘要，这些以前机器做不了的事情，现在都能自动化了。

AI + 自动化工具的组合，简单说就是：**自动化工具负责"搬运"（数据在系统之间流转），AI负责"理解"（语义分析、内容生成、智能判断）。** 两者结合，很多过去需要人盯着做的事情就能自动跑起来。

## 用Zapier / Make / n8n搭建简单工作流

这三个是目前最主流的自动化工作流平台，区别如下：

**Zapier**：最易上手，界面直观，适合非技术人员。免费版有限制（100个任务/月），付费版按量计费。海外工具生态最丰富。

**Make（原Integromat）**：比Zapier灵活，可以做更复杂的逻辑分支。界面是可视化流程图，学习曲线稍陡但上限更高。性价比比Zapier好。

**n8n**：开源可自部署，适合有技术团队的企业。功能最强大，可以高度自定义，但需要一定的技术基础来部署和维护。

如果你是第一次接触，建议从Zapier或Make开始。

### 搭建工作流的通用步骤

**第一步：明确触发条件。** 这个流程从什么事件开始？收到一封邮件？表单提交了？某个时间到了？每周一早上9点？

**第二步：定义处理逻辑。** 触发之后做什么？是直接转发，还是需要先分析内容再决定下一步？如果需要AI介入（比如分析邮件情绪、生成摘要），在这一步加入AI模块。

**第三步：设置输出动作。** 处理完之后，结果发到哪里？发邮件通知？写入Google Sheet？发到钉钉/飞书群？创建一个任务卡片？

**第四步：测试和调整。** 用真实数据跑一遍，看看输出是否符合预期。AI模块的Prompt可能需要调几次才能达到满意的效果。

## 三个常见办公场景自动化示例

### 示例一：新邮件自动分类通知

**场景：** 客服邮箱每天收到大量邮件，需要区分"投诉"、"咨询"、"合作"三类，分别通知对应的负责人。

**工作流设计：**
1. **触发器：** Gmail / Outlook 收到新邮件
2. **AI处理：** 调用ChatGPT API，Prompt如下：
   ```
   请判断以下邮件属于哪个类别：投诉、咨询、合作。只返回类别名称。
   邮件标题：{{邮件标题}}
   邮件正文：{{邮件正文前500字}}
   ```
3. **条件分支：** 根据AI返回的类别，走不同的通知渠道
   - 投诉 → 立即发飞书消息给客服主管，并标红
   - 咨询 → 发到客服组群
   - 合作 → 转发给商务部门
4. **记录：** 所有邮件的分类结果写入一个Google Sheet，便于统计

### 示例二：表单提交自动生成报告

**场景：** 每周五各部门通过在线表单提交周报数据，你需要汇总成一份周报。

**工作流设计：**
1. **触发器：** 设置一个定时任务（每周五下午5点），或者当所有表单都提交后触发
2. **数据收集：** 从表单工具（金数据/飞书表单/Google Forms）拉取本周所有提交
3. **AI处理：** 把所有提交数据发给AI，Prompt如下：
   ```
   以下是各部门本周提交的周报数据，请帮我生成一份团队周报摘要：

   {{各部门的表单数据}}

   周报格式：
   1. 本周整体进展概述（3-5句话）
   2. 各部门亮点（每个部门1句话）
   3. 需要关注的风险和问题
   4. 下周重点工作
   ```
4. **输出：** 将生成的周报发送到团队群/发邮件给管理层

### 示例三：定时数据汇总

**场景：** 每天早上需要一份前一天的关键业务指标汇总。

**工作流设计：**
1. **触发器：** 每天早上8:30
2. **数据获取：** 从数据源拉取前一天的数据（Google Sheet、数据库API、业务系统的报表接口）
3. **AI处理：** 让AI做数据对比和简要分析——跟前一天比、跟上周同期比、有没有异常值
4. **输出：** 格式化为一张卡片消息，发送到管理层的工作群

这三个例子的复杂度是逐渐递增的。第一个最简单，半小时就能搭好；第二个需要一两个小时；第三个可能需要和技术团队配合接数据源。建议从第一个开始练手。

## 企业级AI助手搭建入门

如果你的需求更进阶——比如想搭一个能回答公司内部问题的AI客服，或者一个能检索公司知识库的AI助手——就需要用到AI助手搭建平台。

### 三个主流平台对比

**Coze（扣子）：** 字节跳动出品，免费额度慷慨，支持接入飞书、微信等渠道。中文生态友好，适合国内团队快速搭建AI Bot。内置了插件市场、工作流编排、知识库功能，上手门槛低。

**Dify：** 开源的AI应用开发平台，可以自部署也可以用云服务。灵活度高，支持多种模型（OpenAI、Claude、国内各大模型），适合有一定技术能力的团队。优势在于数据自主可控。

**GPTs（ChatGPT的自定义助手）：** 如果你的团队已经在用ChatGPT Plus/Team，GPTs是最简单的选择。不需要写代码，通过对话配置即可。但局限是只能在ChatGPT生态内使用，无法接入其他渠道。

### 快速上手建议

如果你是第一次搭建AI助手，按这个路径来：

1. **选一个具体的小场景。** 别一上来就想做一个"万能AI助手"，先选一个具体的痛点，比如"帮新同事回答公司制度相关的常见问题"。

2. **准备知识库素材。** 把相关的文档、FAQ、规章制度整理好，上传到平台的知识库功能里。

3. **配置AI助手。** 写好系统提示词（System Prompt），明确告诉AI它的角色、它能回答什么、不能回答什么、回答的风格是什么。

4. **内测和优化。** 先在小范围内让几个同事试用，收集反馈，调整Prompt和知识库内容。典型的问题是"AI回答了知识库里没有的内容"——这时需要加上约束："如果知识库中没有相关信息，请回答'这个问题我还不确定，建议联系HR同事确认'。"

5. **推广使用。** 优化到满意后，接入团队的日常沟通渠道（飞书、钉钉、企业微信），让大家日常使用。

从简单场景切入，先让团队尝到甜头，再逐步扩展AI助手的能力范围。这比一步到位做个大系统要靠谱得多。
