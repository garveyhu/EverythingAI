# 前言


欢迎翻开这份AI 认知升级指南。

先说一句大实话：这份教程不会把你变成AI专家。它做不到，也没打算这么做。

那它能做什么？它能帮你把门推开一条缝，让你看见门后面的世界大概长什么样。至于你要不要走进去、走多远、往哪个方向走——这些事情只有你自己能决定。

## 为什么要写这份教程

> “团队最大的浪费：知识只存在某个人的脑子里。”
> 
> 这是为什么我要写这篇教程的原因。

过去一年，我见过太多这样的场景：有人用ChatGPT写了一封邮件，觉得"哦，AI就是个聊天机器人"；有人听说AI要取代所有人，焦虑得睡不着；也有人觉得这些都是炒作，过几年就没人提了。

这三种人有一个共同点——他们对AI的认知都停留在碎片化的印象上，没有形成系统的理解。

这份教程就是要解决这个问题。不是要让你精通每一个AI工具，而是帮你建立一个基本的认知框架。有了这个框架，你再去接触任何AI产品、任何AI概念，都不会觉得两眼一抹黑。

## 关于"AI复利"这件事

我特别想聊聊一个概念：**AI复利**。

复利大家都懂，银行存钱利滚利。AI的学习也有类似的效应——你今天花一小时搞懂了提示词工程，明天你用AI做每一件事的效率都会提升；你这周学会了用AI辅助写代码，下周你就能腾出时间去研究AI做数据分析。

但这个复利有一个前提：**必须是你自己去探索的**。

看别人的教程、听别人的分享，能给你方向和灵感，但真正改变你思维方式的，是你自己动手试的那些时刻。你亲手写了一个提示词、发现效果不好、改了三遍终于得到想要的结果——这个过程中形成的直觉和判断力，是任何教程都给不了你的。

所以我的建议是：看到任何一个知识点，立刻打开对应的工具试一下。哪怕试出来的结果很烂，那也比光看不练强十倍。

## 六个篇章，一条路径

这份教程分为六个篇章，按照从认知到实践的顺序编排：

**第一篇：AI通识篇。** 这是地基。AI到底是什么、怎么跟它有效沟通、市面上有哪些工具可用、用AI要注意什么红线。不管你是什么岗位，这部分都得过一遍。

**第二篇：设计篇。** 面向设计师和有视觉需求的同事。AI怎么辅助设计流程，从灵感收集到出图到迭代优化。

**第三篇：产品篇。** 面向产品经理和业务人员。AI如何帮你做市场分析、写需求文档、梳理用户故事。

**第四篇：开发篇。** 面向工程师。AI辅助编程的工具链、最佳实践、以及怎么让AI真正提升你的开发效率而不是制造更多bug。

**第五篇：测试篇。** 面向QA同事。AI在测试用例生成、自动化测试、缺陷分析方面的应用。

**第六篇：通用篇。** 不分岗位都用得上的AI技能——用AI做PPT、写文档、处理Excel、辅助学习等等。

你不需要从头读到尾。通识篇建议每个人都看，后面的篇章挑跟自己相关的深入就行。

## 最后一点

不要怕犯错。

用AI的过程中，你会遇到它胡说八道的时候，会遇到它怎么都理解不了你意思的时候，也会遇到它给出的东西让你眼前一亮的时候。这些都是学习的一部分。

没有人一开始就能把AI用得很溜。那些看起来用得很好的人，无非是比你早踩了几个月的坑。

好了，不多说了。翻到下一页，我们开始。
